working_directory

This is the current working directory of the game.

In most cases, this is the same as the program_directory. It may be a different directory when testing your project through the IDE, to point to where the Included Files are stored.

When using this directory to write out a file, GameMaker will redirect it to the game_save_id location if the file system sandbox does not allow writing to the program directory.

It's not recommended to use this variable because of the uncertainty of where it might point. Either use a filename without this path prefixed (see Accessing File Areas), or use program_directory or game_save_id depending on what you need.

Read: The File System

 

Syntax:

working_directory

 

Returns:

String

 

Example:

ini_open(working_directory + "temp_ini.ini");

This will open an INI file from the working directory of the game (creating it if it does not already exist). This could be the save area or the program directory depending on the sandbox level.
